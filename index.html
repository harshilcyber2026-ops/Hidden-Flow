⚙ Reverse Engineering · Challenge #09

Hidden Flow

This binary uses indirect jumps and function pointers to hide its validation logic. No direct calls — only a dispatch table and call rax. Trace every pointer.

Description

The binary accepts a secret key and validates it through five hidden check functions. None are called directly — all calls go through a function pointer dispatch table loaded into a register at runtime, producing call rax instructions in the disassembly.

Static analysis will show no named function calls inside the validation loop. You must reconstruct the dispatch table from the data section, follow each pointer to its target function, understand what each one checks, then derive the secret key.

The key is never stored as a plaintext string. The final check computes a custom rolling hash and compares it against a hardcoded constant.

+0x00push rbp
+0x01mov rbp, rsp
+0x04mov [rbp-0x28], rdi ; tbl = dispatch table ptr
+0x08mov [rbp-0x30], rsi ; idx = table index
+0x0cmov [rbp-0x38], rdx ; s = input string
+0x10mov [rbp-0x40], rcx ; n = length

+0x14mov rax, [rbp-0x28] ; rax = tbl
+0x18mov rdx, [rbp-0x30] ; rdx = idx
+0x1cmov rax, [rax+rdx*8] ; ← load fn ptr from table
+0x20mov [rbp-0x8], rax ; store volatile (prevents inlining)
+0x24mov rax, [rbp-0x8] ; reload
+0x28mov rsi, [rbp-0x40] ; arg2 = n
+0x2cmov rdi, [rbp-0x38] ; arg1 = s
+0x30call rax ; ← INDIRECT CALL — target unknown statically
+0x32pop rbp

Analysis Strategy

Step 1 — Find the table: In Ghidra open the binary. In the Symbol Tree or Defined Data view, look for a block of 5 consecutive 8-byte values in the .data section all pointing into .text. That is DISPATCH[].

Step 2 — Follow each pointer: Double-click each address. Rename the functions (chk_length, chk_digit, etc). Understand the pass condition (return 0 = pass).

Step 3 — Reconstruct the hash: The 5th function compares toy_hash(input, 10) against 0x581d3018. Port the algorithm to Python, then brute-force within the known char set: 10 chars, lowercase + digits + underscore.

Tip: The char set is [a-z0-9_] and length is exactly 10. Combined with the hash, a targeted Python brute-force or angr symbolic execution solves it in seconds.
